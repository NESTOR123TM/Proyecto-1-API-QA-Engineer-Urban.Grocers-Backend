# Proyecto-1-API-QA-Engineer-Urban.Grocers-Backend
Contexto del proyecto Pruebas de una REST API de backend para Urban.Grocers orientadas a validar operaciones CRUD y reglas de negocio. Alcance centrado en calidad funcional, contratos JSON y comportamiento HTTP de endpoints POST, GET, PUT y DELETE.
Análisis
Diseñé y ejecuté más de 30 casos de prueba basados en requisitos funcionales y reglas de negocio. Apliqué técnicas de equivalence partitioning y boundary value analysis. Validé estructuras JSON, códigos HTTP (200/400/404) y manejo de errores. Ejecuté pruebas con Postman y gestioné defectos en Jira con trazabilidad completa.

Conclusiones principales
La API cumplió los flujos principales, pero se detectaron defectos en validaciones de entrada y manejo de errores. Se mejoró la cobertura de reglas de negocio y la robustez de validaciones mediante casos negativos y de frontera.
